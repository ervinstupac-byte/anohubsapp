# AnoHUB: Autonomous Hydro-Intelligence Platform

## State of the Union (v1.0.0 - Cognitive Integration)

**AnoHUB** is a next-generation **Autonomous Hydro-Intelligence Platform**. It transcends traditional control systems by blending advanced 3D digital twins, autonomous cognitive decision-making, and predictive Monte Carlo forecasting to manage Hydroelectric Power Plants (HPP) seamlessly.

### 🌟 Key Ecosystem Features

#### 1. Universal Turbine Hubs (3D Digital Twins)
A centralized ecosystem spanning the three core hydraulic machines:
*   **Francis Hub**: 22 interactive Standard Operating Procedures (SOPs).
*   **Kaplan Hub**: Blade pitch physics and operational optimization.
*   **Pelton Hub**: Jet synchronization and multi-nozzle physics.
*   **Live Physics**: Real-time headers displaying RPM, Grid Frequency (Hz), Active Power (MW), and dynamic stress across all hubs.

#### 2. The Cognitive Core (Advanced Intelligence)
Moving beyond static alerts, the system evaluates real-time market and grid states through an extensive suite of over 110 localized micro-services:
*   **SovereignBrainCore**: A high-speed, deterministic strategy evaluator balancing Grid Support, Eco Compliance, Market Price (MAX_PROFIT), and Machine Health. Automatically recommends operational modes based on real-world constraints.
*   **QuantumForecaster**: An experimental Monte Carlo simulator projecting asset wear 10 years into the future using randomized weather, flood, and drought scenarios to predict the ultimate failure mode and Remaining Useful Life (RUL).
*   **MasterIntelligenceEngine**: Ingests sensor telemetry, runs cross-correlation safety rules, and triggers automated actions on threshold breaches.

#### 3. Reality Bridge & Synapse (The "Golden Thread")
*   **Detection to Ticket**: When an engineer (or the Intelligence Engine) identifies a risk (e.g., Water Hammer > 20 bar), a unified workflow instantly creates an actionable ticket.
*   **Supabase Persistence**: Infinite, cloud-synced persistence for Maintenance Logs and Work Orders. Features Realtime Collaboration and Optimistic UI updates.

#### 4. Enterprise-Grade Foundation
*   **Global Command Palette**: `Ctrl+K` navigation to instantly jump between turbine hubs or analytics modules.
*   **Internationalization (i18n)**: Full support for 6 languages (EN, ES, FR, DE, ZH, BS).
*   **Forensic PDF Engine**: Local-first, highly detailed PDF report generation (`ProfessionalReportEngine.ts`) with embedded custom fonts.

### 🛠 Tech Stack
*   **Frontend**: React, TypeScript, Vite.
*   **3D Engine**: Three.js, React Three Fiber, Drei.
*   **Backend / Persistence**: Supabase (PostgreSQL, Realtime, Auth).
*   **Cognitive Services**: Custom TypeScript evaluation engines.
*   **Styling**: Tailwind CSS (Executive/Dark Industrial Theme).
*   **State Management**: React Context + Zustand + Supabase Subscriptions.

## AnoHUB Forensic Gate — Operation Manual (CI/CD)

### System Architecture Notes
- Canonical dossier manifest: generated by `scripts/generate_library.py` and materialized as `scripts/hashes_applied.json`. The runtime loader builds the canonical library in `src/data/knowledge/DossierLibrary.ts` which exposes `DOSSIER_LIBRARY`, `PATH_INDEX`, and `resolveDossier()` for consumers.  
- All automated actions (test and production) are persisted to the `automated_actions_audit` table. The SQL migration is under `supabase/migrations/001_create_automated_actions_audit.sql`.

### CI Enforcements
- The Forensic Gate is enforced by CI (`.github/workflows/integration-test.yml`). A branch cannot be merged unless:
  - The integrity check passes: `scripts/verify_nc102_integrity.cjs` must confirm `scripts/hashes_applied.json` contains 854 dossiers.  
  - The sensor harness runs: `scripts/sensor_harness.cjs` replays cavitation/surge sequences; the CI must successfully persist an audit row via `scripts/persist_test.cjs` and `scripts/assert_audit_row.cjs`.  

### Domain & Secrets
- GitHub Actions Secrets (required):
  - `SUPABASE_URL` — your Supabase project URL (e.g. `https://<project>.supabase.co`)  
  - `SUPABASE_KEY` — service role key or a key with INSERT/select permissions for `automated_actions_audit`.  

- Vercel / Production env vars (required):
  - `VITE_SUPABASE_URL` — Supabase URL for client-side features.  
  - `VITE_SUPABASE_ANON_KEY` — Supabase anon key for public reads (if used).  

- **No hardcoded keys:** `src/services/supabaseClient.ts` reads credentials only from environment variables. Missing vars fall back to a noop client for offline/CI builds.

   ```powershell

   $env:SUPABASE_URL='https://<your-project>.supabase.co'
   ```
